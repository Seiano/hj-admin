应用采用「域（Domain）+ Schema 引擎」的分层架构：
- 入口编排层 `src/app/`：`App.tsx` 仅挂载 `<BrowserRouter>` → `AppProviders` → `AppRoutes`；`bootstrap.ts` 利用 Vite 的 `import.meta.glob('../domains/*/manifest.ts', { eager: true })` 在构建时自动发现所有域清单，导出 `allManifests` / `getAllRoutes()` / `buildMenuTree()`，实现路由与菜单零注册。
- 路由层 `router.tsx`：遍历 `getAllRoutes()`，有 `schema` 的路由交给 `SchemaPage` 自动渲染，无 schema 则懒加载自定义组件，统一包裹 `MainLayout`。
- 域层 `src/domains/<name>/`：每个域自包含 `manifest.ts`（声明 name/label/icon/menuGroup/order/routes）、`schema.ts`（定义 PageSchema）、`repository.ts`（调用 `registerMockData` 注入 mock）、`pages/`（自定义页面）、`types.ts`、`mock.ts`，新增域只需新建目录并写 manifest。
- 共享基础设施 `src/shared/`：`data/` 提供 `DataProvider` + `useRepository` + `HttpRepository`/`MockRepository`，数据源模式由 `config/domains.config.ts` 集中开关；`schema-engine/` 是核心——`types.ts` 定义 `PageSchema`/`ColumnDef`/`FilterField`/`ModalDef` 等类型，`SchemaPage.tsx` 根据 schema 自动渲染筛选栏、Tab、表格、分页、行/批量/工具栏操作及弹窗，`renderers/` 支持字符串引用渲染器。
- 布局层 `src/layouts/`：`MainLayout` 组合 `Sidebar`（读取 `buildMenuTree()`）与 `Topbar`，内容区通过 `<Outlet />` 渲染路由页面。
依赖方向：app → domains → shared；shared 不反向依赖任何 domain。
