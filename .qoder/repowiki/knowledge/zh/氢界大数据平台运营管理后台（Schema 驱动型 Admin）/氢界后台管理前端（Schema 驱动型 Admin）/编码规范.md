- 每个业务域以独立目录存放，必须包含 `manifest.ts`（声明 name/label/icon/menuGroup/order/routes）、`schema.ts`（PageSchema）、`repository.ts`（`registerMockData` 注入 mock）、`types.ts`、`mock.ts`，新增域无需修改路由或菜单代码。
- 页面以「配置优先」方式编写：列表/表单/弹窗行为全部通过 `PageSchema` 字段声明，仅在需要复杂交互时才手写 `component` 懒加载组件。
- 数据访问通过 `useRepository(entityKey)` 获取，实体 key 与 `domainConfig` 中的域名一一对应，切换 `mock`/`http` 仅需改 `config/domains.config.ts`。
- 列渲染器同时支持内联函数与字符串引用（`render: 'xxx'`），字符串形式配合 `renderers/index.ts` 注册表复用跨域渲染逻辑。
- 路由项通过 `hideInMenu: true` 隐藏于侧边栏但保留导航能力（如编辑详情页），菜单树由 `buildMenuTree()` 从 manifest 自动生成并按硬编码分组顺序输出。
