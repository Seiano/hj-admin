应用以 `src/app/bootstrap.ts` 为启动编排点：利用 Vite 的 `import.meta.glob('../domains/*/manifest.ts', { eager: true })` 在构建期扫描所有域清单，按 `order` 排序后输出 `allManifests`，再由 `router.tsx` 聚合为全局路由、由 `buildMenuTree()` 组装侧边栏菜单；新增域只需在 `src/domains/<name>/manifest.ts` 中声明路由与菜单项，无需改动注册代码。

数据层通过 `src/shared/data/DataProvider.tsx` 提供统一上下文，依据 `src/config/domains.config.ts` 中每个域的 `mock | http` 模式，在运行时注入 `MockRepository` 或 `HttpRepository` 实例；各域 `repository.ts` 在模块加载时调用 `registerMockData(domainName, data)` 向 DataProvider 注册本地 mock 数据，从而让 `useRepository(domain)` 在页面中零配置获取数据源。

共享能力集中在 `shared/`：`schema-engine/` 提供 `SchemaPage`、渲染器与类型定义，被所有域复用；`data/` 提供 Repository 抽象、Context 与 useHook；`layouts/` 提供 MainLayout/Sidebar/Topbar 等布局壳。业务域仅暴露 `index.ts` 作为入口，内部按 manifest → schema → repository → pages 分层组织，彼此通过约定好的接口契约解耦。
